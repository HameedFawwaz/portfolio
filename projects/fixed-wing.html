Fixed-Wing Pitch Control System (SITL)

Design, implementation, and validation of a cascaded pitch control system for a fixed-wing UAV using software-in-the-loop simulation.

Overview

This project focused on the development of a longitudinal pitch control system for a fixed-wing unmanned aerial vehicle. The objective was to achieve stable and responsive pitch angle tracking while respecting actuator limits and maintaining robustness to modeling uncertainty.

All development and testing were performed in a software-in-the-loop (SITL) environment, enabling rapid iteration, repeatability, and safe validation prior to any hardware deployment.

System Architecture

A cascaded control architecture was selected, consisting of an outer-loop pitch angle controller and an inner-loop pitch rate controller. This structure allows the inner loop to handle fast dynamics while the outer loop regulates slower attitude behavior.

Control Design

Both the pitch angle and pitch rate controllers were implemented using PID control laws. The inner-loop rate controller was tuned first to achieve a fast, well-damped response, after which the outer-loop angle controller was tuned around the closed inner loop.

  • Cascaded PID structure
  • Inner-loop bandwidth significantly higher than outer loop
  • Anti-windup to handle actuator saturation

Controller gains were tuned iteratively in simulation using step responses and disturbance rejection tests, with emphasis on stability margins and smooth actuator behavior.

Simulation Environment

The control system was evaluated in a software-in-the-loop environment that simulated vehicle dynamics, sensor feedback, and actuator response. This setup enabled repeatable testing across a range of flight conditions.

  • Fixed-wing longitudinal dynamics model
  • Simulated sensor noise and delays
  • Real-time control loop execution

Validation & Results

Validation focused on pitch angle tracking performance, transient response, and robustness to disturbances. Step inputs and reference changes were used to evaluate settling time, overshoot, and steady-state error.

  • Stable pitch tracking across commanded setpoints
  • Acceptable rise time and minimal overshoot
  • Controlled actuator activity within limits

Engineering Tradeoffs

Key tradeoffs included balancing responsiveness against actuator saturation, as well as selecting controller gains that remained robust under modeling uncertainty. A more aggressive tuning improved tracking performance but increased sensitivity to noise and saturation effects.

Lessons Learned

  • Cascaded architectures simplify tuning and improve robustness
  • Inner-loop performance strongly constrains outer-loop behavior
  • SITL enables safe, rapid iteration before hardware testing
